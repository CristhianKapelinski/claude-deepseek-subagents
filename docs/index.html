The Problem

Claude Code is brilliant — but every subagent it spawns (searching, reading dozens of files, mechanical edits, running tests) also burns Claude. That is like paying senior-engineer rates for intern work.

On a Pro or Max plan, you hit your weekly usage limit fast. On API billing, you pay Claude prices for work a cheaper model would do just as well.

The Key Differentiator

ANTHROPIC_BASE_URL is global, so the naive belief is "a proxy breaks my subscription." This proxy only rewrites auth for deepseek-* requests. For claude-* requests it forwards Claude Code's own headers untouched — so your subscription OAuth token reaches Anthropic exactly as if the proxy were not there.

Install

One command
curl -fsSL https://raw.githubusercontent.com/CristhianKapelinski/claude-deepseek-subagents/main/install.sh | bash

Then drop your DeepSeek key (from platform.deepseek.com) into ~/.claude/deepseek/.env (chmod 600):

.env
DEEPSEEK_API_KEY=sk-...

Requirements: Node.js 18+, curl, and claude (Claude Code) logged into your subscription.

Use

Terminal
claude-ds        # instead of `claude`

Claude Code exactly as usual — except when the orchestrator delegates, the work runs on DeepSeek. Plain claude stays completely untouched. Switch anytime.

Watch it happen from another terminal:

Manager
ds-proxy status      # up/down, pid, key present
ds-proxy log         # live — every call: model -> DEEPSEEK | anthropic
ds-proxy stop        # NEVER from inside claude-ds

⚠ Warning: Never run ds-proxy stop or ds-proxy restart inside a claude-ds session — that session talks to the API through the proxy, so killing it drops its own connection. Manage the proxy from a normal terminal.

Multi-Agent Workflows

Workflow agent() calls inherit the session model (Claude) by default. The injected policy tells the orchestrator to pass agentType: 'ds-flash' / 'ds-pro' (or model: 'deepseek-v4-flash') to fan-out agents, so the bulk of the work lands on DeepSeek while orchestration and final verification stay on Claude.

FAQ

Does my main agent really stay on my subscription?
Yes. claude-* requests are forwarded to api.anthropic.com with their headers untouched — the same OAuth token Claude Code already holds. The proxy never sees or needs an Anthropic API key.
Is this a fork or a patch of Claude Code?
No. It is a local proxy plus two standard ~/.claude/agents/*.md files. Uninstall and everything is back to stock.
Will it break when Claude Code updates?
The subagent/model parts are official features. OAuth passthrough through a local proxy is tolerated, not officially supported — if an update ever changes auth handling, just run plain claude. Nothing here is destructive.
Does prompt caching still work?
Yes, per provider: Anthropic caches the main agent's context; DeepSeek caches within a subagent's loop. Caches do not cross the provider boundary (inherent, not a limitation of the proxy).
What about DeepSeek's count_tokens?
DeepSeek's Anthropic-compatible endpoint may not implement it, so the proxy synthesizes a rough estimate for deepseek-* routes — preflight never stalls.
How is OAuth passthrough handled?
The proxy forwards claude-* headers untouched to Anthropic. This is tolerated, not officially supported. If a Claude Code update changes auth, just use plain claude until things settle.
